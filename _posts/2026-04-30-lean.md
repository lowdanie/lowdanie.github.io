# Introduction

[Lean](https://lean-lang.org/) is a [functional](https://lean-lang.org/functional_programming_in_lean)
programming language whose type system is powerful enough to encode mathematical proofs.
One implication is that it is possible to use `lean` both to write programs and to
construct mathematical proofs about a programs properties.

The goal of this post is to introduce `lean` by using it to implement insertion sort 
and prove that the output is always sorted.

As a warm up, we'll define the `LessThan` relation on natural numbers and prove basic 
properties such as transitivity.

Next we'll define the `IsLowerBound` and `IsSorted` list properties which are needed
to state and prove properties of insertion sort.

Finally, we'll implement insertion sort and prove that its output is always sorted.

# Dependent Types

Lean bridges the gap between programs and proofs via
[dependent types](https://lean-lang.org/functional_programming_in_lean/Programming-with-Dependent-Types/#Functional-Programming-in-Lean--Programming-with-Dependent-Types).

To see how this works, we'll use dependent types to define a `LessThan` relation
on natural numbers and prove some of its properties.

## Overview

In this section we'll introduce the `LessThan` dependent type and give a high level 
overview of the relationship between dependent types and proofs.

The specifics of the `lean` syntax will be provided in the following section.

The `LessThan` type represents the "less than" relation between pairs of natural numbers.
It is defined as follows:

```lean
inductive LessThan : Nat → Nat → Prop where
  | base : (x : Nat) → LessThan x (x + 1)
  | step : (x : Nat) → (y : Nat) → LessThan x y → LessThan x (y + 1)
```

At a high level, this means that `LessThan` defines a _family_ of `Prop` types parameterized by
pairs of natural numbers. Objects of types in the family can be built recursively 
via the `base` and `step` constructors.

For example, we can use the `base` constructor to build an element of type `LessThan 0 1`:

```lean
LessThan.base 0 : LessThan 0 1
```

We can then use the `step` constructor to build an element of type `LessThan 0 2`:

```lean
LessThan.step 0 1 (LessThan.base 0) : LessThan 0 2
```

We call `LessThan` a _dependent type_ because each pair of natural numbers `(x y : Nat)`
defines a different type `LessThan x y`.

The `Prop` in the definition means that types in this family (such as `LessThan 0 1`)
belong to special class of types called _propositions_. In `lean` a proposition type is
defined to be _true_ if there exists an object of that type and _false_ if not.
An object in a `Prop` type is called a _proof_ of the proposition.

For example, `LessThan 0 1` is a proposition and we can write this explicitly as

```lean
LessThan 0 1 : Prop
```

Furthermore, this proposition is true and the proof is the object `LessThan.base 0`
we constructed above:

```lean
LessThan.base 0 : LessThan 0 1
```

We can declare the relationship between the proposition and the proof more explicitly
using the `theorem` keyword:

```lean
theorem zero_lt_one : LessThan 0 1 := LessThan.base 0
```

As an example of a false proposition, consider

```lean
LessThan 0 0 : Prop
```

Indeed, it is easy to see that neither of
the available constructors of `LessThan` can output an object of type `LessThan 0 0`.
For example the `LessThan.base` constructor always outputs objects of type `LessThan x x + 1`
for some natural number `x : Nat`.
And there does not exist a natural number `x` such that `x = x + 1 = 0`.


## Lean Syntax

In the previous section we defined the `LessThan` type

```lean
inductive LessThan : Nat → Nat → Prop where
  | base : (x : Nat) → LessThan x (x + 1)
  | step : (x : Nat) → (y : Nat) → LessThan x y → LessThan x (y + 1)
```

In this section we'll dive into the details of the syntax.

### Functions

In `lean`, if `S` and `T` are types then
the expression `S → T` is the type representing functions from `S` to `T`.
The arrow `→` is associative to the right. For example: 

```lean
Nat → Nat → Nat = Nat → (Nat → Nat)
```

In other words, `Nat → Nat → Nat` represents functions `f` that take a natural number
`n : Nat` and return a _function_ 

```lean
f(n) : Nat → Nat
```

This means that we can use `f` to map a pair of natural numbers `n m : Nat` to an
output in `Nat` via:

```lean
f(n)(m) : Nat
```

The process of chaining together functions of a single variable to represent multi-variable
functions is called
[currying](https://en.wikipedia.org/wiki/Currying).

The upshot is that we can think of the type

```lean
S → T → U
```

As representing functions that take two arguments with types `S` and `T` and output
an object of type `U`.
